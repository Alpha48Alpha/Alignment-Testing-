Prompt Attack Lab

Attack Tests prompt-attack-lab/jailbreak-tests.md

Adversarial jailbreak prompts that attempt to bypass model safety policies. Includes instruction override attacks and roleplay-based bypass attempts, each with an evaluation goal and expected safe behavior.

Attack Tests prompt-attack-lab/prompt-injection-tests.md

Prompt injection test cases that attempt to inject malicious instructions into model context (e.g., hidden override commands embedded within translation tasks). Includes expected safe behavior for each case.

Defense prompt-attack-lab/defense-strategies.md

Recommended mitigations against prompt attacks: input validation, instruction hierarchy enforcement, output monitoring, and prompt isolation techniques to separate system instructions from user input.

Evaluation — Hallucination & Safety Metrics

Metrics evaluations/metrics.md

Defines the Hallucination Rate metric in full: how to identify unsupported claims, cross-reference against reliable sources, and assign a 0–3 quality score (3 = no hallucinations, 0 = substantially hallucinated response).

Prompts prompts/technical/hallucination_rate.md

Technical hallucination detection prompts: biography fabrication, invented scientific claims, citation hallucination, legal reference errors, and plausible-but-false historical claims — each with hallucination signals to watch for.

Framework evaluations/framework.md

Evaluation framework covering all five safety-relevant dimensions. Describes consistency measurement methodology: run all prompt variants, score each, compute score variance — a consistent prompt set has variance ≤ 0.5.
